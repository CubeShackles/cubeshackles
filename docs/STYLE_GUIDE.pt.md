# Guia de Estilo — Português

[English](STYLE_GUIDE.en.md) | [Português](STYLE_GUIDE.pt.md)

**Proprietário: CubeShackles (liderado pelo fundador).**

Português formal, institucional, adequado a Angola e ao público lusófono
mais amplo. Este guia complementa
[`GLOSSARY.en-pt.md`](GLOSSARY.en-pt.md) — a terminologia controlada tem
prioridade sobre preferências de estilo individuais.

1. **Verbos precisos para o grau de maturidade.** Usar "implementa" /
   "disponibiliza" / "ativo" apenas para o que existe e é exercitado por
   testes. Usar "concebido para" / "destinado a" / "estruturado" para direção
   comprometida com estrutura em disco. Usar "planeado" apenas quando o
   repositório ainda não existe. Ver
   [`LOCALIZATION_POLICY.md`](LOCALIZATION_POLICY.md) §4.
2. **A tradução nunca reforça uma afirmação.** Se o inglês diz "protótipo", o
   português diz "protótipo" — nunca "implementado".
3. **Português europeu/angolano, não brasileiro.** "Utilizador" (não
   "usuário"), "ficheiro" (não "arquivo"), "equipa" (não "time"), "registo"
   (não "registro"), "controlo" (não "controle" quando referente a
   `plano de controlo`).
4. **Evitar coloquialismos e linguagem de marketing.** Frases declarativas
   curtas, adequadas a um leitor institucional (banco central, comissão de
   mercado de capitais, investidor, auditor) — não a um visitante de página
   de aterragem.
5. **Preservar identificadores.** Código-fonte, comandos, variáveis de
   ambiente, nomes de ficheiros, nomes de pacotes, rotas de API, campos de
   esquema, nomes de eventos, classes, funções, constantes de protocolo,
   terminologia criptográfica, nomes de produto e marca registada, nomes de
   entidade legal, símbolos de token e códigos de moeda — nunca traduzir.
6. **Acrónimos.** Definir na primeira utilização (ex.: "KYC (Conheça o Seu
   Cliente)"), depois usar apenas o acrónimo, tal como no inglês.
